Установка системы «Дело» средствами графического инсталлятора. Работа системы «Дело» с кластером СУБД, когда второй экземпляр БД находится в режиме «только для чтения». Автоматизированный переход без потери данных и прерывания действующих сессий пользователей системы «Дело» на резервный экземпляр системы. Корректное завершение запущенных процессов (фоновых задач) в основном экземпляре системы и их перезапуск в резервном экземпляре. Масштабирование работы фоновых задач за счет использования нескольких хостов для распределения нагрузки. Резервирование файлового хранилища. Параллельная индексации объектов системы. Для каждого индексируемого объекта автоматически создается несколько параллельных потоков индексации. Формирование отчетных форм на втором (реплицированном) экземпляре сервера БД. Мониторинг группы серверов с помощью инструмента ELK (Elastic Search, Logstash, Kibana).
